Local Jenkins Host

jenkins-infra

A reusable Jenkins server repo for local macOS development. It hosts the Jenkins runtime, bootstrap scripts, and operator documentation, while application repos keep only their own code, tests, and Jenkinsfile.

Role In The Demo

This repo is the infrastructure side of the hackathon story. It keeps Jenkins reusable and separate, so the BSF app repo can focus on application logic, Codex skill assets, and its own pipeline.

Infrastructure Responsibilities

  • Jenkins bootstrap scripts for local macOS use
  • Jenkins plugins and Configuration as Code files
  • operator-facing docs for setup and recovery
  • an optional container-based backup path

Primary Path

Start Jenkins On This Laptop

cd /Users/anil/Documents/anil/jenkins-infra
chmod +x scripts/*.sh
./scripts/start-jenkins-local.sh

Jenkins runs directly on macOS with a repo-local JENKINS_HOME. This keeps the setup easy to move and easy to demo.

Integration Flow

How It Connects To hackathon-ci-demo

1

Create A Pipeline Job

Point Jenkins at https://github.com/anilneeluru09/hackathon-ci-demo.git using Pipeline script from SCM.

2

Track main

Use branch */main and script path Jenkinsfile.

3

Run The BSF Pipeline

The app repo executes ./ci/run_bsf_backend_tests.sh, which bootstraps Java, Maven, and MongoDB inside the repo workspace.

4

Publish Results

Jenkins publishes JUnit results from reports/junit/*.xml and archives generated report artifacts.

Polling Model

Why pollSCM Is Used

  • the Jenkins server is running on a laptop
  • GitHub cannot call a webhook on localhost
  • pollSCM('H/2 * * * *') keeps demo triggering simple
  • new commits are detected roughly every two minutes
